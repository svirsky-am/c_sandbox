# TCP клиент-сервер на примере управления банковскими ячейками


## Проверяем сервер
Для проверки TCP-соединений можно обойтись без клиента и использовать консольную утилиту netcat. Вот её официальное описание:


NAME (ИМЯ)

nc — произвольные TCP- и UDP-соединения и прослушивание

DESCRIPTION (ОПИСАНИЕ)

Утилита nc (или netcat) используется практически для всего, что связано с TCP или UDP.
Она умеет:

устанавливать TCP-соединения,

отправлять UDP-пакеты,

слушать произвольные TCP- и UDP-порты,

выполнять порт-сканирование,

работать как с IPv4, так и с IPv6.

В отличие от telnet(1), утилита nc хорошо подходит для написания скриптов, и выводит сообщения об ошибках в стандартный поток ошибок (stderr), а не в стандартный поток вывода (stdout), как это делает telnet(1) в некоторых случаях. 


## Запускаем сервер 
```sh 
make i_2_3_task_vault
```
или 
```sh
cargo run -p i_2_3_task_vault
```

В соседнем терминале запустите netcat. Вы увидите приглашение для отправки команд:
```sh
nc 127.0.0.1 7878 
```
Вставляем команлды
```sh
PUT 1 gold 30
PUT 5 money 100 
LIST
GET 1
GET 5
PUT 5 ring 10
GET 2
EXIT
BYE
```
